# 近视风险检测功能集成说明

## 📝 功能概述

基于`pose_detector.py`的近视风险检测原理，成功在`pose_detection_thread.py`中集成了近视风险检测功能，并在UI界面上添加了相应的显示和控制组件。

## 🎯 主要改进

### 1. 配置文件更新 (`config.py`)

新增了以下配置项：
```python

    # ... 原有配置 ...

    'consecutive_myopia_risk_threshold': 3,  # 连续检测到近视风险多少次后发出警告
```

### 2. 后台检测线程增强 (`pose_detection_thread.py`)

#### 新增信号
- `myopia_risk_alert`: 近视风险警告信号

#### 新增属性
- `consecutive_myopia_risk`: 连续近视风险次数计数

#### 新增方法
- `check_myopia_risk()`: 检查近视风险并发送警告
- 更新了`get_detection_statistics()`方法，包含近视风险统计

#### 检测流程增强

- **近视风险提醒开关**: 用户可以开启/关闭近视风险警告
- **近视风险阈值设置**: 可调整连续检测到近视风险多少次后发出警告

## 🔍 近视风险检测原理

### 检测方法
- **距离计算**: 通过计算鼻子位置到图片底部的距离来评估眼部与屏幕的距离
- **阈值判断**: 当距离小于`MYOPIA_RISK_THRESHOLD`(200像素)时，判定存在近视风险

### 风险等级分类
- **高风险**: 距离 < 160px (阈值的80%)
- **中风险**: 160px ≤ 距离 < 180px (阈值的90%) 
- **低风险**: 180px ≤ 距离 < 200px
- **无风险**: 距离 ≥ 200px

### 警告机制
- **连续检测**: 当连续检测到近视风险达到阈值次数时发出警告
- **自动重置**: 当检测到无风险状态时，自动重置连续计数
- **独立监控**: 近视风险检测与低头检测并行工作，互不干扰

## 🚀 使用方法

1. **启动系统**: 运行`python start_posture_monitor.py`
2. **配置设置**: 在UI界面中可以：
   - 开启/关闭近视风险提醒
   - 调整近视风险警告阈值(1-10次)
3. **实时监控**: 系统会实时显示：
   - 当前近视风险等级
   - 近视风险统计信息
   - 健康建议
4. **警告响应**: 当检测到连续近视风险时会弹出警告对话框

## 🎉 功能特点

- **无缝集成**: 与原有低头检测功能完全兼容
- **实时监控**: 持续监控用户眼部与屏幕的距离
- **智能分级**: 根据风险程度提供不同级别的警告
- **用户友好**: 提供直观的UI界面和清晰的健康建议
- **可配置**: 用户可以根据需要调整检测参数

这次功能增强成功地将静态的近视风险检测功能转化为了动态的实时监控系统，为用户提供了更全面的健康保护。
