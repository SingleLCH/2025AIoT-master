# 批量批改功能说明

## 功能概述

批量批改功能是专为老师设计的智能作业批改工具，支持批量拍摄学生作业并进行AI智能分析，生成详细的教学报告和建议。

## 功能特点

### 1. 智能拍照
- 复用现有拍照逻辑，确保稳定性
- 仅限学校模式使用，无需人脸识别
- 支持多张作业照片的连续拍摄
- 每次拍摄前自动清理照片文件夹

### 2. AI智能分析
- 集成home_work.py的批量分析功能
- 使用阿里云通义千问QVQ视觉推理模型
- 自动识别常见错误和薄弱知识点
- 生成个性化教学建议

### 3. 数据库存储
- 分析结果保存到指定数据库
- 数据库：`poem.e5.luyouxia.net:28298`
- 库名：`data_info`
- 表名：`studentinfo`
- 用户：`root`，密码：`Wsad1234+`

### 4. 美观的结果展示
- 分层级显示分析结果
- 包含错误分析、薄弱知识点、教学建议
- 响应式HTML布局，易于阅读

## 使用方法

### 1. 进入功能
1. 启动应用并选择"学校模式"
2. 在功能选择页面点击"批量批改"图标
3. 系统将自动初始化摄像头和处理器

### 2. 手势指令控制
- **6-0-1**: 拍摄作业照片
  - 可以连续多次拍摄
  - 每次拍摄都会显示在摄像头预览页面
  
- **6-0-2**: 返回功能选择页面
  - 随时可以退出当前流程
  - 会自动清理已拍摄的照片
  
- **6-0-3**: 上传并分析照片
  - 开始AI智能分析流程
  - 分析完成后自动保存到数据库
  - 显示详细的分析报告

### 3. 流程说明
1. **第一步：拍摄作业** - 使用6-0-1拍摄需要分析的作业照片
2. **第二步：AI分析** - 使用6-0-3上传照片进行智能分析
3. **第三步：结果展示** - 自动显示分析结果和教学建议

## 技术架构

### 核心组件
- `BatchHomeworkHandler`: 批量批改处理器，负责整个流程控制
- `BatchHomeworkPage`: 批量批改页面，提供用户界面
- `home_work.py`: AI分析引擎，调用通义千问模型
- `DatabaseHandler`: 数据库操作，扩展支持批量批改数据

### 数据流程
```
拍照 -> 文件存储 -> AI分析 -> 结果解析 -> 数据库保存 -> 界面展示
```

### 数据库表结构
```sql
CREATE TABLE studentinfo (
    id INT AUTO_INCREMENT PRIMARY KEY,
    time DATETIME,
    analysis_result TEXT,    -- 完整分析结果JSON
    common_mistakes TEXT,    -- 常见错误JSON
    weak_points TEXT,        -- 薄弱知识点JSON
    teaching_advice TEXT     -- 教学建议JSON
);
```

## 注意事项

### 使用限制
- 仅在学校模式下可用
- 需要确保网络连接稳定（用于AI分析）
- 需要摄像头权限

### 性能考虑
- 建议单次分析照片数量不超过10张
- AI分析可能需要几十秒时间，请耐心等待
- 大量照片分析时建议分批进行

### 故障排除
1. **摄像头问题**: 检查摄像头连接和权限
2. **网络问题**: 确保能访问阿里云API
3. **数据库连接**: 验证数据库配置和网络连通性
4. **分析失败**: 检查照片质量和格式
