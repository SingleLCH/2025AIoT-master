# 低头检测与近视风险评估系统说明

## 🎯 系统概述

这个智能检测系统通过分析人体姿态特征点，实现**双重健康风险评估**：

1. **低头检测**：通过鼻子与肩膀连线的位置关系判断低头状态
2. **近视风险检测**：通过鼻子到图片底部的距离评估近视风险

该系统能够全面评估用户的坐姿健康和用眼健康，提供科学的健康建议。

## 🔬 检测原理

### 低头检测算法
1. **关键点检测**：使用YOLOv11模型检测人体关键点
2. **位置分析**：计算鼻子(关键点0)与肩膀连线的垂直距离
3. **状态判断**：
   - 鼻子**高于**肩膀连线 → 正常姿态
   - 鼻子**低于**肩膀连线 → 低头状态

### 近视风险检测算法
1. **距离测量**：计算鼻子到图片底部边缘的像素距离
2. **阈值比较**：与配置的阈值（默认200像素）进行比较
3. **风险判断**：
   - 距离 ≥ 200px → 无近视风险
   - 160px ≤ 距离 < 200px → 低风险
   - 120px ≤ 距离 < 160px → 中风险  
   - 距离 < 120px → 高风险

### 风险等级分级
**低头程度分级：**
- **轻微低头**：低头程度 < 0.15
- **中度低头**：0.15 ≤ 低头程度 < 0.3
- **严重低头**：低头程度 ≥ 0.3

**近视风险分级：**
- **无风险**：鼻子到底部距离 ≥ 200px
- **低风险**：160px ≤ 距离 < 200px
- **中风险**：120px ≤ 距离 < 160px
- **高风险**：距离 < 120px

## 🔍 YOLOv11关键点说明

系统基于YOLOv11模型检测17个人体关键点（COCO格式）：

```
0: 鼻子        9: 右手腕      
1: 左眼        10: 左臀部     
2: 右眼        11: 右臀部     
3: 左耳        12: 左膝盖     
4: 右耳        13: 右膝盖     
5: 左肩膀      14: 左脚踝     
6: 右肩膀      15: 右脚踝     
7: 左手肘      16: 左脚趾（未使用）
8: 右手肘      17: 右脚趾（未使用）
```

**核心检测点：**
- **关键点0（鼻子）**：低头检测和近视风险评估的核心参考点
- **关键点5、6（左右肩膀）**：肩膀连线计算，判断低头状态
- **关键点3、4（左右耳朵）**：姿态分析中的辅助参考点

## 🎯 坐姿健康分析详细指标

### 分析维度

1. **头部前倾角度**
   - 测量鼻子相对于耳朵的角度偏移
   - 健康范围：< 15°
   - 轻微问题：15° - 30°
   - 严重问题：> 30°

2. **肩膀水平度**
   - 检测左右肩膀的高度差
   - 健康范围：< 5°
   - 轻微倾斜：5° - 10°
   - 明显倾斜：> 10°

3. **颈部对齐度**
   - 计算头部相对于肩膀中线的偏移比例
   - 健康范围：< 0.15
   - 轻微前倾：0.15 - 0.3
   - 严重前倾：> 0.3

### 健康等级评定

- **优秀** (85-100分)：坐姿标准，无明显问题
- **良好** (70-84分)：基本健康，有轻微问题
- **一般** (50-69分)：存在多个问题，需要注意
- **需要改善** (0-49分)：坐姿不良，需要立即改正

## 🔧 故障排除

### 常见问题

1. **"未检测到人体"**
   - 确保图片中人体上半身完整
   - 检查图片清晰度和光线条件
   - 尝试降低置信度阈值

2. **"关键点置信度较低"**
   - 改善图片质量和拍摄角度
   - 确保面部和肩膀清晰可见

3. **安装问题**
   - 确保已正确安装所有依赖：`pip install -r requirements.txt`
   - 检查Python版本兼容性
   - 确保网络连接正常

4. **分析结果异常**
   - 检查是否为正面坐姿照片
   - 确认图片中只有一个主要人物
   - 验证拍摄角度是否合适

### 获取最佳结果的建议

- 📸 使用正面、水平视角拍摄
- 💡 确保充足、均匀的光线
- 👤 保持自然的坐姿状态
- 🎯 让头部和肩膀完全可见
- 📏 保持适当的拍摄距离

### 技术要求
1. 首次运行会自动下载YOLOv11模型（约6MB）
2. 确保图片格式正确且文件存在
3. 需要稳定的网络连接（首次下载模型）
4. 建议Python 3.8+版本
