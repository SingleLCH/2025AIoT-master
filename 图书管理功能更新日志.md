## 版本 1.0.1 (2024-12-13)

### 🐛 问题修复

#### 问题1: 6-0-2信号无法返回到功能选择界面

**问题描述**:
- 用户在图书管理界面使用6-0-2指令时，无法返回到功能选择页面
- 特别是在人脸识别失败后，系统状态被重置，导致后续的MQTT指令无法处理

**根本原因**:
1. 当人脸识别失败时，`book_management_handler.py`中的`_reset_process()`方法会将`is_processing`设为`False`
2. `handle_mqtt_command`方法中的检查逻辑要求`is_processing`为`True`才处理指令
3. 这导致6-0-2返回指令被忽略，用户无法退出界面

**修复方案**:
1. **优先处理返回指令**: 在`book_management_handler.py`中修改`handle_mqtt_command`方法，让6-0-2指令无论系统状态如何都优先处理
2. **改进主程序路由**: 在`main.py`中增强指令路由逻辑，确保图书管理页面的指令能正确转发
3. **页面检测机制**: 添加当前页面检测，确保在图书管理页面时指令被正确处理

**代码变更**:

`book_management_handler.py`:
```python
# 修改前
if not self.mqtt_ready or not self.is_processing:
    return

# 修改后  
# 6-0-2（返回）指令始终处理，无论流程状态如何
if command == '6-0-2':
    self._handle_back_command()
    return
    
# 其他指令只有在流程启动时才处理
if not self.mqtt_ready or not self.is_processing:
    return
```

`main.py`:
```python
# 添加页面检测
if (self.book_management_page and current_widget == self.book_management_page):
    if self.book_management_handler:
        self.book_management_handler.handle_mqtt_command(command)
        return

# 改进处理器检查
if command == '6-0-2' or self.book_management_handler.is_processing:
    self.book_management_handler.handle_mqtt_command(command)
    return
```

#### 问题2: 图书管理界面显示返回按钮

**问题描述**:
- 用户要求不在图书管理界面的头部显示返回按钮
- 应该只通过MQTT指令（6-0-2）来返回

**修复方案**:
移除`book_management_page.py`中头部的返回按钮，用占位符保持布局平衡

**代码变更**:
```python
# 修改前
back_btn = QPushButton("返回")
# ... 按钮配置和连接 ...
header_layout.addWidget(back_btn)

# 修改后
header_layout.addStretch()  # 占位符，保持布局平衡
```

2. **状态无关**: 返回操作不依赖于系统的内部状态
3. **用户友好**: 确保用户在任何情况下都不会被困在某个界面

**影响范围**:
- 仅影响图书管理功能
